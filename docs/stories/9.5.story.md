# Story 9.5: Suíte de testes (Vitest + Testing Library + Playwright e2e)

## Story

**As a** desenvolvedor do EA360,
**I want** Vitest, Testing Library e Playwright configurados com exemplos executáveis,
**so that** todos os épicos tenham uma suíte de testes funcional disponível para escrever testes unitários, de componentes e end-to-end.

## Acceptance Criteria

1. Vitest instalado e configurado (`vitest.config.ts`); `npm run test` executa a suíte sem erros de configuração.
2. `@testing-library/react` e `@testing-library/user-event` instalados; ao menos um teste de componente de exemplo passa (ex: renderização do Button do shadcn/ui).
3. Playwright instalado e configurado (`playwright.config.ts`); `npm run test:e2e` executa ao menos um teste e2e de exemplo que navega para `localhost:3000` e verifica que a página carrega.
4. Script `npm run test` definido em `package.json` e executa Vitest.
5. Script `npm run test:e2e` definido em `package.json` e executa Playwright.
6. Diretório de testes unitários: `tests/unit/`; diretório de testes e2e: `tests/e2e/`.
7. `jsdom` configurado como ambiente de testes no Vitest para testes de componente React.
8. `npm run lint` e `npm run typecheck` passam após a configuração (arquivos de config tipados corretamente).

## Dev Notes

- Deliverable D-051; Épico E-XACC; PRD §7 "vitest + @testing-library/react, playwright (e2e)".
- Esta story cria a infraestrutura de testes que as demais stories (especialmente 9.1 e 9.2) referenciam.
- [AUTO-DECISION]: usar `jsdom` como ambiente Vitest (não `happy-dom`) por ser mais maduro e amplamente suportado pelo ecossistema Testing Library.
- [AUTO-DECISION]: instalar apenas Chromium para Playwright no MVP (não Firefox/Safari) para reduzir tamanho do download e tempo de CI.
- [AUTO-DECISION]: `globals: true` no Vitest para evitar imports explícitos de `describe`/`it`/`expect` em cada arquivo de teste, alinhado ao padrão Jest familiar.
- `tests/unit/` para Vitest; `tests/e2e/` para Playwright — conforme Story 1.1 Dev Notes (referência a "Story 9.5" para framework de testes).
- O teste e2e requer que a aplicação esteja rodando — no CI, usar `webServer` config do Playwright para subir `npm run dev` automaticamente.

### Testing

- A própria story é a suíte de testes — critério de Done: `npm run test` e `npm run test:e2e` executam sem erros de configuração.
- Teste unitário exemplo: `tests/unit/components/button.test.tsx` — renderiza `<Button>` e verifica que está no DOM.
- Teste e2e exemplo: `tests/e2e/home.spec.ts` — navega para `localhost:3000` e verifica `page.title()` ou presença de elemento no `<body>`.
