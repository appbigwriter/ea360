## Story

**As a** empreendedor com um menu personalizado gerado,
**I want** que o sistema distribua meu orçamento nos canais recomendados seguindo a regra 70/20/10 (núcleo/crescimento/experimento),
**so that** tenha uma carteira de monetização balanceada entre segurança e crescimento.

## Acceptance Criteria

1. Server Action `allocateBudget(recommendationId, totalBudget, ratios?)` distribui o orçamento entre as faixas de risco.
2. Faixas padrão: núcleo 70% / crescimento 20% / experimento 10%; parametrizáveis via `ratios` (validado: soma = 100%).
3. Canais são classificados por `risk_band` com base no `risk_score` do GOM: score 1–2 = core, 3 = growth, 4–5 = experiment.
4. Verba de cada faixa é distribuída igualmente entre os canais da faixa (ou proporcional ao match_score).
5. Resultado persistido em `allocations` + `allocation_items` com valores de `amount`, `percentage` e `risk_band`.
6. Validação: orçamento total = soma de `allocation_items.amount` (tolerância ±R$0,01).
7. Retorna erro descritivo se não houver canais suficientes em alguma faixa.
