## Story

**As a** product manager do EA360,
**I want** que o sistema registre automaticamente quando um usuário completa a Entrevista 360 e gera um menu personalizado,
**so that** seja possível medir a taxa de ativação (% de usuários que completam a Entrevista e geram um menu) conforme métrica §10 do PRD.

## Acceptance Criteria

1. Evento de analytics `interview_completed` é disparado quando o usuário finaliza todos os passos da Entrevista 360 e o perfil de monetização é persistido.
2. Evento de analytics `menu_generated` é disparado quando a função de matchmaking (`fn_match_channels`) produz pelo menos um resultado e a recomendação é salva na tabela `recommendations`.
3. Ambos os eventos incluem `business_id`, `user_id`, `timestamp` e `session_id` como propriedades obrigatórias.
4. A taxa de ativação (usuários com `menu_generated` / usuários com `interview_completed`) é calculável a partir dos dados registrados.
5. A instrumentação não bloqueia o fluxo principal — falhas no registro de analytics não propagam erros para o usuário.
6. `npm run lint` e `npm run typecheck` passam sem erros após a implementação.
