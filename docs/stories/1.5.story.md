## Story

**As a** administrador da plataforma EA360,
**I want** que todas as tabelas de negócio tenham políticas RLS que isolem dados por `business_id` ou `owner_id`,
**so that** nenhum usuário consiga ler ou modificar dados de outro negócio.

## Acceptance Criteria

1. RLS habilitado (`ALTER TABLE ... ENABLE ROW LEVEL SECURITY`) em todas as tabelas que armazenam dados de negócio.
2. Policy de SELECT em cada tabela filtra por `business_id = (SELECT id FROM businesses WHERE owner_id = auth.uid())` ou equivalente direto.
3. Policies de INSERT/UPDATE/DELETE garantem que o `auth.uid()` seja dono do negócio associado.
4. Tabelas do catálogo público (`gom_pillars`, `gom_categories`, `gom_channels`) têm policy de SELECT pública (sem auth) e DML restrito a service role.
5. Teste manual com dois usuários distintos confirma isolamento: usuário A não acessa dados de usuário B.
6. Migrações de RLS aplicadas com `supabase db push` sem erros.
