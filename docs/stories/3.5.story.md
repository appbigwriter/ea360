## Story

**As a** empreendedor que completou a Entrevista 360,
**I want** que o sistema gere e salve um perfil de monetização estruturado a partir das minhas respostas,
**so that** o matchmaking e o alocador possam usar este perfil para me recomendar canais adequados.

## Acceptance Criteria

1. Ao completar a entrevista (`status = 'completed'`), sistema chama LLM para gerar um perfil de monetização estruturado (JSON).
2. Perfil gerado é salvo em `monetization_profiles` com `interview_id` (FK → `interviews`) e `business_id`.
3. Perfil é um objeto JSON consultável com campos: `objectives`, `philosophy`, `current_stage`, `resources`, `excluded_channels` (IDs de canais a evitar pelo filtro de filosofia).
4. Tabela `monetization_profiles` com campos: `id`, `business_id`, `interview_id`, `profile_data` (JSONB), `created_at`, `updated_at`.
5. Usuário é redirecionado para `/app/profile` após geração com visualização resumida do perfil.
6. Se LLM falhar, perfil é criado com os dados brutos das respostas e flag `is_llm_generated = false`.

#### Schema do `profile_data` (AC3) — contrato para o matchmaking (E-MENU)

Definido em `src/lib/monetization/profile.ts` (`MonetizationProfile`). Persistido em `monetization_profiles.profile_data` (JSONB). Todos os campos sempre presentes (defaults vazios no fallback):

```jsonc
{
  "objectives":        ["string", ...], // camada 1 — objetivos & metas
  "philosophy":        ["string", ...], // camada 2 — valores e o que recusa
  "current_stage":     "string",        // camada 3 — caixa/margem/audiência/risco
  "resources":         ["string", ...], // camada 4 — tempo/time/energia/capital
  "excluded_channels": ["string", ...]  // IDs/slugs de canais a evitar (preenchido de fato pela Story 3.6 — filtro de filosofia)
}
```

No fallback (AC6, sem LLM), `profile_data` também inclui `raw_answers` (`{question, answer, layer}[]`) com as respostas cruas para auditoria/reprocessamento. As colunas estruturadas legadas da tabela (`goals`/`horizon`/`risk_tolerance`/`philosophy_excludes`/...), usadas por `fn_channel_score`/`fn_match_channels`, permanecem intactas e não são tocadas por esta story.
