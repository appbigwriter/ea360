## Story

**As a** desenvolvedor do EA360,
**I want** o cliente Supabase configurado para SSR com `@supabase/ssr` e as variáveis de ambiente validadas,
**so that** todas as chamadas ao banco de dados e autenticação possam ser feitas de forma segura tanto no servidor quanto no cliente.

## Acceptance Criteria

1. Pacotes `@supabase/supabase-js` e `@supabase/ssr` instalados.
2. Utilitário `createClient` para Server Components configurado em `src/lib/supabase/server.ts`.
3. Utilitário `createBrowserClient` para Client Components configurado em `src/lib/supabase/client.ts`.
4. Middleware Next.js (`middleware.ts`) implementado para refresh de sessão Supabase em cada request.
5. Variáveis `NEXT_PUBLIC_SUPABASE_URL` e `NEXT_PUBLIC_SUPABASE_ANON_KEY` validadas na inicialização; app falha com mensagem clara se ausentes.
6. `npm run build` conclui sem erros relacionados a Supabase.
7. Supabase CLI listado nos pré-requisitos do projeto (`README` ou `docs/guides/`).
