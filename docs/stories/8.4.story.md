## Story

**As a** usuário final que conversa com o bot WhatsApp do empreendedor,
**I want** que o bot se identifique sempre como bot e que eu possa acionar um atendente humano,
**so that** minha experiência seja transparente e eu não seja enganado sobre com quem estou falando.

## Acceptance Criteria

1. Todo fluxo de bot gerado inclui identificação automática: primeira mensagem do bot inclui texto declarando que é um bot automatizado.
2. Texto de identificação usa o `bot_disclosure` definido no funil (Story 8.1) — obrigatório.
3. Todos os templates do funil incluem opção de "Falar com humano" (botão CTA ou palavra-chave como "HUMANO").
4. Quando usuário aciona escape humano, o fluxo sinaliza "handoff para humano" e para o bot (sem resposta automática).
5. Configuração de escape humano salva no JSON do funil (campo `human_escape_config`).
6. UI de configuração permite definir: palavra-chave de escape, mensagem de handoff ("Transferindo para nossa equipe...").
