## Story

**As a** desenvolvedor do EA360,
**I want** TypeScript strict mode, ESLint e Prettier configurados e funcionando como parte do pipeline de desenvolvimento,
**so that** o código mantenha consistência, tipagem segura e qualidade automática ao longo de todo o projeto (AIOX Article V — Quality First).

## Acceptance Criteria

1. TypeScript configurado com `"strict": true` no `tsconfig.json`; `npm run typecheck` (ou `tsc --noEmit`) passa sem erros.
2. ESLint configurado com regras para TypeScript e Next.js; `npm run lint` passa sem erros em todos os arquivos do projeto.
3. Prettier configurado com arquivo `.prettierrc` (ou equivalente); `npm run format` formata o código sem conflitos com ESLint.
4. Script `npm run lint` está definido em `package.json` e executa ESLint em `src/` (ou diretório equivalente).
5. Script `npm run typecheck` está definido em `package.json` e executa `tsc --noEmit`.
6. Script `npm run format` está definido em `package.json` e executa Prettier.
7. Arquivo `.eslintrc` (ou `eslint.config.mjs`) está presente e integra `eslint-config-next` e regras TypeScript.
8. Arquivo `.prettierignore` está presente para excluir `.next/`, `node_modules/` e arquivos gerados.
