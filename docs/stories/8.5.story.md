## Story

**As a** desenvolvedor do EA360,
**I want** a extensão `pgvector` habilitada e a tabela `oracle_documents` criada com suporte a embeddings,
**so that** o Oráculo de conformidade Meta possa armazenar e buscar documentos por similaridade vetorial.

## Acceptance Criteria

1. Extensão `pgvector` habilitada no projeto Supabase (`CREATE EXTENSION IF NOT EXISTS vector`).
2. Tabela `oracle_documents` criada: `id`, `title`, `content` (text), `url` (text), `embedding` (vector(1536)), `source_type` (enum: meta_policy/help_center/api_docs), `scraped_at`, `created_at`, `updated_at`.
3. Índice HNSW ou IVFFlat criado em `oracle_documents.embedding` para busca por similaridade.
4. Função de busca por similaridade `match_oracle_documents(query_embedding vector, match_threshold float, match_count int)` definida.
5. RLS: documentos são públicos para leitura (todos os usuários autenticados); INSERT restrito a service role.
6. Migração aplicada sem erros.

## Dev Notes

- PRD §5: "pgvector para o Oráculo".
- PRD §8 tabela: `oracle_documents`.
- PRD F4.2: "Oráculo RAG — `pgvector`, ingestão".
- Vector dimension: 1536 (compatível com `text-embedding-3-small` da OpenAI, PRD §7: `openai` SDK).
- HNSW é mais rápido para busca; IVFFlat é melhor para grandes datasets. Para MVP, HNSW.
- `<=>` = distância cosseno (0 = idêntico, 2 = oposto).
- Depende de Story 1.4 (schema base).
