## Story

**As a** empreendedor,
**I want** criar uma conta e fazer login no EA360,
**so that** meu perfil e dados de negócio fiquem associados à minha identidade de forma segura.

## Acceptance Criteria

1. Fluxo de cadastro por e-mail/senha via Supabase Auth funcionando.
2. Fluxo de login por e-mail/senha via Supabase Auth funcionando.
3. Logout limpa sessão e redireciona para página pública.
4. Tabela `profiles` criada (vinculada a `auth.users`) com campos `id`, `full_name`, `avatar_url`, `updated_at`.
5. Tabela `businesses` criada com campos `id`, `owner_id` (FK → `profiles.id`), `name`, `created_at`.
6. Trigger `set_updated_at()` aplicado às tabelas com `updated_at`.
7. RLS base habilitado: usuário só lê/edita seus próprios `profiles` e `businesses`.
8. Rotas protegidas redirecionam para login quando não autenticado (via middleware do Story 1.2).
9. UI de login e cadastro acessível em `/auth/login` e `/auth/signup`.
