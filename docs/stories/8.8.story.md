## Story

**As a** sistema EA360,
**I want** um pipeline automatizado que re-scrapeie periodicamente as páginas de política da Meta e atualize os embeddings do Oráculo,
**so that** as respostas do Oráculo reflitam sempre a documentação mais recente e o risco de conformidade desatualizada seja eliminado.

## Acceptance Criteria

1. Edge Function Supabase (`supabase/functions/oracle-refresh/index.ts`) implementada com cron de re-scrape.
2. Função usa `@mendable/firecrawl-js` para scrapeiar URLs de política da Meta configuradas.
3. Conteúdo scrapeado é comparado com o existente em `oracle_documents`; documentos alterados são re-embeddados.
4. `oracle_documents.scraped_at` é atualizado a cada re-scrape.
5. Cron configurado para execução semanal (ou conforme configuração).
6. Métrica de frescor: `MAX(now() - scraped_at)` de todos os documentos deve ser < 8 dias.
7. Erros de scrape são logados sem quebrar a Edge Function (tolerância a falhas de rede).
8. `FIRECRAWL_API_KEY` configurada como secret da Edge Function; nunca exposta no cliente.
