## Story

**As a** product manager do EA360,
**I want** que o sistema registre quando um usuário salva uma alocação de portfólio e quando utiliza o Oráculo (Q&A RAG),
**so that** seja possível medir o valor percebido (% que salva uma alocação; uso recorrente do Oráculo) conforme métrica §10 do PRD.

## Acceptance Criteria

1. Evento de analytics `allocation_saved` é disparado quando o usuário persiste uma alocação na tabela `allocations`.
2. Evento de analytics `oracle_queried` é disparado quando o usuário realiza uma consulta Q&A no Oráculo (RAG).
3. O evento `allocation_saved` inclui `business_id`, `user_id`, `allocation_id`, `total_budget` e `timestamp`.
4. O evento `oracle_queried` inclui `business_id`, `user_id`, `query_length` (número de caracteres, sem expor o texto completo) e `timestamp`.
5. Uso recorrente do Oráculo é rastreável: é possível contar quantas consultas `oracle_queried` cada `user_id` fez em um período.
6. Falhas no registro de analytics não propagam erros para o usuário (fire-and-forget).
7. `npm run lint` e `npm run typecheck` passam sem erros após a implementação.
